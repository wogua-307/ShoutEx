# 输出全靠吼小游戏介绍文档

## 一句话介绍

《输出全靠吼》是一款用声音控制角色、用嗓门完成挑战的像素街机小游戏合集。玩家不再只靠手指点击，而是通过喊、稳住音量、爆发尖叫、控制收声来推动角色行动，在短时间内获得直接、好笑、容易传播的游戏反馈。

## 吸引人简介

晚上：啊啊啊啊啊啊啊啊啊！

《输出全靠吼》是一款专治憋屈的声控像素小游戏。用你的嗓门开车、飞天、施法、发泄。喊得越大，反馈越狠。生活已经够压抑了，至少这一局，输出真的全靠吼。

在公司当了一天“情绪稳定的成年人”，消息回不完，需求改不完，会议开不完，连崩溃都要挑个不影响别人的时间。

别憋了，来《输出全靠吼》。

这里不考 KPI，不看汇报，不拼手速，也不问你为什么还没做完。你只需要对着手机：啊啊啊啊啊啊啊啊啊！

喊得越稳，赛车跑得越远；喊得越猛，火箭飞得越高；喊得越准，小鸟穿过障碍；喊得够狠，直接打出爆炸伤害。

它不是让你真的解决生活里的所有压力，但至少能给你一个 30 秒的出口：把憋住的话、没发出去的火、没来得及叹的气，都变成游戏里的分数、伤害、高度和距离。

## 游戏简介

这是一款围绕“声音输入”设计的微信小游戏。游戏把麦克风音量转化为角色移动、速度、蓄力、攻击和技能释放，让玩家用真实声音参与游戏。

整体体验是轻量、快节奏、强反馈的像素电玩城。玩家进入大厅后，可以选择不同声控玩法：让小鸟随音量飞行、用声音驱动赛车冲刺、喊出峰值发射火箭、累计分贝打出爆发伤害，或拖动法师瞄准并用不同音量释放法术。每局时间短、规则直观，适合碎片时间、朋友围观、聚会整活和社交分享。

## 核心定位

- 产品类型：声控互动小游戏合集
- 平台形态：微信小游戏
- 视觉风格：像素街机、霓虹高对比、复古游戏厅
- 体验关键词：好上手、强反馈、轻竞技、整活传播、短局挑战
- 核心差异：把“声音大小、持续、峰值、收声”变成可玩的操作维度

## 核心卖点

### 1. 声音就是操作杆

传统小游戏通常依赖点击、滑动、长按。《输出全靠吼》把麦克风音量变成核心输入，玩家喊得大、喊得稳、喊得准都会影响结果。

这带来两个优势：

- 操作新鲜：第一次玩就能感受到“我真的在用声音控制游戏”。
- 围观有趣：旁边的人能直接听到玩家在“输出”，天然适合分享和线下互动。

### 2. 五种玩法对应五种声音能力

游戏不是简单把同一种音量条套进不同皮肤，而是让每个小游戏对应一种不同的声音挑战：

- 尖叫小鸟：考验音量高低控制和收声节奏。
- 十秒狂飙：考验持续稳定输出。
- 声波火箭：考验短时间峰值爆发。
- 分贝发泄馆：考验峰值和平均声量的综合爆发。
- 言出法随小法师：考验手指瞄准和音量分档施法。

### 3. 短局强反馈

每个玩法都控制在几秒到几十秒内完成，结果立刻可见：

- 分数
- 距离
- 高度
- 伤害
- 防守得分

这种结构适合微信小游戏的使用场景：点开快、理解快、失败后马上再来一把。

### 4. 像素街机风格强化记忆点

项目采用深色背景、像素描边、高对比霓虹色、硬边按钮和方块角色。视觉上不追求写实，而是强调“游戏厅感”和“轻松整活感”。

这个风格有几个好处：

- Canvas 绘制成本低，资源包可控。
- 角色和 UI 在小屏幕上识别度高。
- 和“喊出来”的夸张玩法气质匹配。

### 5. 内置设置和历史记录

游戏支持：

- 麦克风权限引导
- 背景音乐开关
- 声控灵敏度调节
- 游戏音量调节
- 单个玩法历史记录
- 最高分记录

这些功能让小游戏从一次性 demo 更接近可持续迭代的产品形态。

## 当前玩法说明

### 尖叫小鸟

玩家通过声音控制小鸟高度。声音越大，小鸟飞得越高；收声后，小鸟会平稳下降。玩家需要在管道间穿梭，尽可能获得更高分数。

设计重点：

- 用声音大小映射目标高度，而不是简单做“喊一下上升”。
- 收声反馈更快，方便玩家避开上方障碍。
- 碰撞盒做了适度宽容，减少“看起来没撞却死了”的挫败感。

适合场景：

- 作为首页主推玩法。
- 新用户快速理解声控机制。
- 做无限挑战和分数分享。

### 十秒狂飙

玩家在 10 秒内持续发声，声音越稳，赛车跑得越远。玩法强调耐力和稳定输出，不只看瞬间大喊。

设计重点：

- 时间短，容易复玩。
- 成绩用距离表达，直观可比较。
- 适合朋友之间比“谁更稳、谁更能喊”。

### 声波火箭

玩家在 3 秒内喊出最大音量峰值，火箭根据峰值自动飞行。玩法强调爆发力和瞬间峰值。

设计重点：

- 先蓄力，后观看结果，节奏清晰。
- 高度结果适合做夸张动画和分享文案。
- 对新手非常友好，不需要复杂操作。

### 分贝发泄馆

玩家在 5 秒内持续输出声音，系统根据峰值和平均声量计算伤害，制造“打碎/爆破”的爽感。

设计重点：

- 偏解压玩法。
- 结果用伤害数字表达，反馈强。
- 适合做情绪释放、下班解压、聚会挑战。

### 言出法随小法师

玩家用手指上下拖动法师进行瞄准，同时用音量决定法术强度：小声释放飞弹，中声释放火球，大声释放陨石。敌人从右侧进攻，玩家需要守住基地。

设计重点：

- 声音负责“火力强度”，手指负责“位置策略”。
- 相比纯声控，玩法更有操作深度。
- 适合作为后续扩展成长系统、关卡系统的核心玩法。

## 目标受众

### 1. 休闲小游戏用户

这类用户希望快速开始、规则简单、失败成本低。小游戏合集形式可以降低单一玩法疲劳。

### 2. 学生和年轻用户

“靠吼操作”天然有整活属性，适合宿舍、课间、朋友聚会等场景。

### 3. 亲子和家庭娱乐用户

玩法不依赖复杂按键，孩子和家长都能理解。火箭、赛车、小鸟等主题也有较强通用性。

### 4. 解压类用户

分贝发泄馆这类玩法可以承接“喊出来、释放一下”的情绪需求，适合做轻量解压方向。

### 5. 社交挑战用户

分数、距离、高度、伤害都适合晒图或发起挑战，例如“你能喊到多少米”“谁的嗓门更强”。

## 视觉与交互风格

### 像素街机大厅

首页使用像素卡片承载玩法入口，尖叫小鸟作为主推大卡，其他玩法组成 2x2 卡片区。卡片边框、图标、难度星级和 START 提示共同形成游戏厅选择界面。

### 小幅动态反馈

首页图标支持随机轻量动效，每 1-2 秒只激活少量图标，避免所有元素一起动造成视觉噪声。

### 原生 Canvas UI

项目使用微信小游戏 Canvas 原生绘制，不依赖 React 打包、不使用 WebView。这样可以更好控制包体、性能和小游戏运行环境。

### 安全区适配

顶部按钮、返回按钮、设置按钮、历史按钮已考虑状态栏和微信胶囊区域，尽量避免遮挡。

## 竞争优势

### 操作方式差异明显

声控输入比常规点击小游戏更容易形成第一印象。用户不需要读很多说明，只要看到“输出全靠吼”，就能理解核心趣味。

### 适合传播

玩家喊叫本身就是内容。线下有人围观，线上可以分享成绩。玩法和名字有梗，容易形成“试一下”的动机。

### 玩法可扩展

声控可以继续扩展出更多模式：

- 合唱协作
- 双人分贝对抗
- 节奏跟唱
- 音高控制
- 关卡闯关
- Boss 战
- 每日挑战

### 技术实现轻量

当前角色、图标和 UI 大多由 Canvas 绘制，音频资源数量可控，不依赖大量贴图。后续可以在包体和性能之间保持灵活。

## 潜在风险

### 1. 场景噪音影响体验

公共环境、背景音乐、人声干扰都可能影响音量识别。需要继续优化噪音门限、灵敏度、校准流程。

### 2. 用户不好意思大喊

声控玩法有天然场景限制。需要提供低音量也能玩的模式，以及“轻声也能控制”的设置。

### 3. 麦克风权限转化

小游戏第一次请求麦克风权限可能让用户犹豫。首页需要清楚说明用途，并在开始挑战前再请求权限，降低突兀感。

### 4. 长时间喊叫疲劳

玩法应保持短局，避免让用户持续高强度喊太久。高强度玩法建议控制在 3-10 秒。

## 优化方向

### 短期优化

- 继续打磨尖叫小鸟的音量响应曲线。
- 给声控防守法师增加敌人分层和不同轨道威胁。
- 增加每个玩法的结算文案，例如“嗓门王者”“稳定输出型选手”。
- 结算页增加分享引导。
- 增加首次进入的轻量教学动画。
- 优化低音量环境下的灵敏度默认值。

### 中期优化

- 增加每日挑战和排行榜。
- 增加成就系统，例如“首次破 10 分”“火箭突破 1000m”。
- 增加角色皮肤和像素装饰。
- 给每个玩法增加更明确的成长目标。
- 增加更多声音维度：音高、节奏、持续时间、爆发次数。
- 增加多人轮流挑战模式。

### 长期优化

- 做成完整的声控派对小游戏平台。
- 增加 PVE 关卡和 Boss 战。
- 引入节奏玩法和音乐玩法。
- 支持用户生成挑战规则。
- 结合活动运营做节日主题玩法。

## 总结

《输出全靠吼》的核心价值不是“把小游戏换成麦克风输入”，而是把声音本身做成游戏机制。它有明确的传播点、轻量的玩法结构、可持续扩展的声控系统，以及适合微信小游戏生态的短局挑战节奏。

后续优化应围绕三件事展开：

- 让声控更准、更稳、更有反馈。
- 让每个玩法都有独立的策略和爽点。
- 让成绩、过程和喊叫本身更容易被分享。
